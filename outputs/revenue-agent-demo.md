## 1. Market Signal Brief

### Demand Pattern
"Giant/jumbo teddy bear" sits in the DTC gifting & novelty-plush category. It is a long-running, recognizable niche in the dropshipping/e-commerce idea space. No live search was run this session, so seasonal-spike claims (Valentine's Day, anniversaries, Christmas) are inference, not verified figures. The product is retail-established (Amazon, Etsy, Costco, Walmart at multiple price points) — a mature, low-novelty niche rather than an emerging one.

Two audiences matter: (1) end consumers who buy the bears — emotional/occasion driven, and (2) the video's actual audience (FuturIQ viewers, aspiring DTC/AI-systems entrepreneurs) — driven by "can I copy this niche with AI" curiosity.

### Audience Psychology
The content audience wants a low-competition niche idea they can execute fast with AI doing the grunt work. They fear the niche is oversaturated guru-bait. They secretly hope AI gives them an unfair speed advantage. Sophistication: intermediate-to-skeptical — distrust unless backed by visible proof.

### Opportunity Score: 24/35
- Urgency: 2/5
- Willingness to pay: 3/5
- Searchability: 4/5
- Clickability: 5/5
- Credibility: 2/5
- Business fit: 4/5
- Under-15-min viability: 4/5

### Risk
Cliché dropshipping niche. Generic treatment reads as recycled guru-bait unless anchored in real proof (screenshots, numbers, live build).

### Strategic Recommendation
- Pitch this as an AI-workflow speed demonstration using an intentionally "boring"/cliché product to prove the system works on anything — not as "here's a hot niche."
- The offer must be the AI build system/workflow itself, not the teddy bear business.
- Insist on a visible artifact (live store, generated ad creative, supplier outreach) — credibility is the weak link.
- Lean into the scale-contrast visual (giant bear vs. person) for clickability.
- Urgency (2/5) is the biggest gap — the CTA needs to manufacture urgency since the niche itself has none.

## 2. Offer Architecture

### Positioning
I help overwhelmed, AI-curious coaches, consultants, and solo entrepreneurs turn any raw business idea — even a boring, saturated one like giant teddy bears — into a positioned, sellable offer in under 15 minutes, without hiring a strategist or learning prompt engineering, by building a reusable Claude Code "Revenue Agent Team" that runs market research, offer design, content angles, and follow-up automatically. The teddy bear build is the stress test, not the business.

### Core Offer
The **Revenue Agent Team** template — the multi-agent Claude Code system that just did this work live on screen.
- Front-end: template download (agent config files + setup guide), one-time purchase.
- Mid-tier: live cohort workshop, installing and running the agent team on the viewer's own idea.
- Back-end: done-with-you AI Systems Build for the entrepreneur's real business.

### Unique Mechanism
**The Signal-to-Offer Pipeline** — a chain-of-custody agent architecture, not a single clever prompt. Each agent has a structured, inspectable output; credibility comes from showing the file trail / pipeline running live, not from claims.

### Value Ladder
Free video -> Lead magnet (Starter Kit) -> Workshop/template purchase -> Premium done-with-you build.

### Proof Requirements
- Live terminal session with a visible timer.
- Real generated output shown in full, not paraphrased.
- File/memory structure visible on screen.
- Before/after split: one-line idea vs. finished offer.
- Ideally a fast second idea run to prove it generalizes.

### Objection Map
- "Teddy bears is a dumb niche" -> address head-on, it's boring on purpose to prove the system.
- "This is just prompt engineering" -> show the multi-agent architecture + memory/file trail.
- "AI tools change every month" -> position as a durable workflow/architecture, not a brittle prompt trick.
- "I'm not technical enough" -> no code, just structured prompts and markdown, beginner-friendly by design.
- "Is this just AI content-for-content's-sake?" -> every output terminates in a revenue-facing artifact.
- "I can grab this template anytime, no rush" (urgency is the real weak spot) -> CTA must manufacture urgency structurally (capped cohort, launch-window pricing, first-N-viewers framing).

## 4. Conversion System

### Lead Magnet: "The Revenue Agent Team Starter Kit"
**Promise:** "The exact 4-agent system you just watched build a sellable offer for giant teddy bears in 14 minutes — set up on your business by tonight."

**Contents:**
1. The four agent configuration files used in the video, ready to drop into a Claude Code project.
2. A 10-minute working setup guide.
3. The full annotated teddy bear run — every output from this document, with commentary on why each agent made the call it made.
4. A one-page "swap-in" worksheet to drop in the viewer's own business idea.

**Why now:** Viewers just watched it happen live; the "would this work on MY business" question is only burning for a few minutes after the video ends.

### In-Video CTA (13:00-14:30, word for word)
"Fourteen minutes. That's positioning, an offer, a content angle, and a follow-up sequence — for giant teddy bears. Look, the teddy bears were never the point. The point is this: the money isn't in the tools, it's in the system that connects them. This exact agent team — the same four agents you just watched work — is what builds offers for real businesses in under fifteen minutes. I put the whole thing in a kit: the agent files, the setup guide, and a full breakdown of every decision they made on this run, so you can see the reasoning, not just the output. It's called the Revenue Agent Team Starter Kit. Link's below — grab it, drop it into Claude Code, and run it on your actual business before you close this tab. I'm keeping it free for the first wave of people who grab it this week, so go now."

### Follow-Up Sequence
1. **Day 0** — "Your Revenue Agent Team Starter Kit (+ the teddy bear breakdown)" — deliver instantly, point to annotated breakdown. CTA: run Agent 1 on your own business today.
2. **Day 2** — "What the teddy bear run got wrong" — build trust via transparency, tease the workshop as where rough edges get fixed live. CTA: reply if your first run felt off.
3. **Day 4** — "Template vs. system — the difference that matters" — kit gets you 80% there, workshop gets you to a sellable offer with live feedback. CTA: join the waitlist.
4. **Day 6** — "Cohort doors close Friday" — structural urgency, capped seats, launch-window pricing. CTA: grab a remaining seat.
5. **Day 9** — "For the businesses that need more than a template" — introduce the done-with-you AI Systems Build. CTA: apply.

### Sales Path
Video -> Lead Magnet (Starter Kit) -> Mid-tier (live cohort workshop) -> Back-end (done-with-you AI Systems Build). Each rung sells the next by exposing the next problem (DIY works but is unpolished -> steering the agents is a skill -> some people want it built for them), not by upselling harder.

### Friction Check
- In-video CTA must not mention pricing or the workshop by name — only the free kit.
- The annotated breakdown must genuinely explain reasoning, not read as a sales document.
- Urgency claims (cohort close date, "first wave free") must stay consistent across all assets or the premium positioning erodes.
- Email 2's "what it got wrong" must read as coaching, not a disclaimer that undercuts the 14-minute claim.
- Email 5 needs a visible criterion for who it's sent to, so the done-with-you offer doesn't feel like a random upsell.
